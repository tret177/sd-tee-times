# SD Tee Times — Project Map

Two pieces, flat in this directory:

```
sd_tee_times.html             ← the dashboard (open in browser)
scraper.py                    ← Python scraper for the four SD muni courses
refresh.sh                    ← wrapper that runs the scraper, writes times.json
alert.py                      ← diffs scrapes and pushes phone notifications
alerts.json                   ← your watcher config (which slots you care about)
com.tret177.teetimes.plist    ← launchd agent that runs refresh.sh every 15 min
requirements.txt
inspect_endpoint.md           ← debugging recipe if the scraper breaks
times.json                    ← generated by refresh.sh; read by the dashboard
.times.prev.json              ← previous scrape (kept for alert diffing)
.refresh.log                  ← combined stdout/stderr of the launchd agent
```

## How they fit together

```
   ┌────────────────────────┐                ┌──────────────────────┐
   │  foreupsoftware.com    │◄──HTTPS────────│  scraper.py          │
   │  (ForeUp booking API)  │                │  (Python)            │
   └────────────────────────┘                └──────────┬───────────┘
                                                        │
                                              writes    ▼
                                            ┌──────────────────────┐
                                            │  times.json          │
                                            └──────────┬───────────┘
                                                       │
                                              fetch()  ▼
                                            ┌──────────────────────┐
                                            │  sd_tee_times.html   │
                                            │  (browser)           │
                                            └──────────────────────┘
```

The dashboard loads `times.json` from the same directory. If the file is
there and is for the date you've selected, the four muni courses show
**LIVE** data (real prices, real spot counts). Every other course (resorts,
daily-fee) stays simulated until you write scrapers for them too.

## Quick start

```bash
# install
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# scrape the next 14 days, 4 players (default)
./refresh.sh

# scrape a custom window (integer N = next N days)
./refresh.sh 7

# scrape a single specific date (any non-numeric arg is treated as ISO date)
./refresh.sh 2026-05-20 2

# write times.json to a different directory
DASHBOARD_DIR=/some/other/dir ./refresh.sh
```

The scraper also accepts these flags directly:

```bash
./.venv/bin/python scraper.py --days 14 --players 4 --json > times.json
./.venv/bin/python scraper.py --dates 2026-05-23,2026-05-30,2026-06-06 --json
```

Serve the directory and open the dashboard:

```bash
python3 -m http.server 8765        # in this folder; leave it running
open http://localhost:8765/sd_tee_times.html
```

You need the HTTP server because browsers block `fetch('./times.json')` from
`file://` URLs (CORS), so double-clicking the HTML directly will always show
"no times.json found · simulated data." Serving over `localhost` fixes it.

The dashboard auto-snaps its date picker to whatever date `times.json` was
scraped for; look for the green **● LIVE** badges on Mission Bay and Balboa
cards.

## What the scraper does

The scraper covers **five public courses across two booking vendors**:

| Course | Vendor | Notes |
|---|---|---|
| Mission Bay | ForeUp | Standard (0-7d) + Advance (8-90d) |
| Balboa Park (18-hole) | ForeUp | Standard + Advance |
| Torrey Pines North + South | ForeUp | Non-Resident (0-90d) per course. Resident classes return 401 — gated by city Resident ID Card |
| Coronado Municipal | TeeItUp (Kenna backend) | Public rates only; resident pricing requires login |

Both vendors expose public JSON APIs that don't require authentication.
The scraper calls each vendor's endpoint once per relevant unit per date,
merges the results, and writes one `times.json` with `tee_times[]` entries
shaped like:

```json
{
  "course_key": "mission_bay",
  "course_name": "Mission Bay Golf Course",
  "date": "2026-05-13",
  "time": "13:30",
  "price": 40.0,
  "resident_price": null,
  "non_resident_price": null,
  "available_spots": 4,
  "holes": 18,
  "booking_url": "https://foreupsoftware.com/index.php/booking/19346/1469#/teetimes",
  "classes": [...],
  "raw": [...]
}
```

If you're a resident with a verified ID card, real Torrey resident prices
won't show up here without adding a login flow — that's deliberate scope.

## Why a static JSON file instead of a server?

Two reasons. One, the original goal was an HTML artifact (no backend). Two,
this is the right architecture for personal use — a cron job writes
`times.json`, the browser reads the latest on page load. No server, no API
costs, no auth.

If you outgrow it (phone access, push notifications, sharing with friends),
the migration is short: push `times.json` to an S3 bucket, point the
dashboard at the public URL, done.

## Background refresh

A launchd agent (`com.tret177.teetimes.plist`) runs `refresh.sh` every 15
minutes while you're logged in. Install with:

```bash
cp com.tret177.teetimes.plist ~/Library/LaunchAgents/
launchctl load -w ~/Library/LaunchAgents/com.tret177.teetimes.plist
```

Logs go to `.refresh.log` next to the scripts. See the plist header for
useful `launchctl` commands.

## Phone alerts

After each scrape, `alert.py` compares the new `times.json` against the
previous one (`.times.prev.json`) and pushes a notification to your phone
via [ntfy.sh](https://ntfy.sh) for any slot that *newly* matches one of
your "watchers" (i.e. matched now, did not match in the previous scrape).

### One-time setup

1. Install the ntfy app: [iOS](https://apps.apple.com/us/app/ntfy/id1625396347) /
   [Android](https://play.google.com/store/apps/details?id=io.heckel.ntfy)
2. Subscribe to your private topic name (visible in `alerts.json`
   under `ntfy.topic`). The default name was randomly generated for you
   so it's effectively a shared secret — don't share it. Change it any
   time by editing `alerts.json` and resubscribing in the app.
3. Edit `alerts.json` to add one or more `watchers` (next section).

### Watcher schema

Each watcher is a JSON object inside the `watchers` array. All fields
except `name` are optional — omit a field to leave that dimension
unconstrained.

| Field | Type | Meaning |
|---|---|---|
| `name` | string | Required. Shows up in the notification title. |
| `course_keys` | list of string | Any of `mission_bay`, `balboa`, `torrey_north`, `torrey_south`. |
| `max_price` / `min_price` | number | Green-fee range (USD). Slots without a price are skipped. |
| `min_spots` | int | Minimum `available_spots`. |
| `earliest_time` / `latest_time` | "HH:MM" | 24-hour. Inclusive on both ends. |
| `days_of_week` | list of "Mon".."Sun" | Filter by weekday of the scraped date. |
| `priority` | "min"/"low"/"default"/"high"/"max" | ntfy notification priority. |

### Example: edit `alerts.json` to taste

```json
{
  "ntfy": {
    "server": "https://ntfy.sh",
    "topic": "sdtee-..."
  },
  "watchers": [
    {
      "name": "Weekend morning Torrey",
      "course_keys": ["torrey_north", "torrey_south"],
      "max_price": 200,
      "earliest_time": "06:00",
      "latest_time": "10:30",
      "days_of_week": ["Sat", "Sun"],
      "min_spots": 2,
      "priority": "high"
    },
    {
      "name": "Cheap twilight any course",
      "max_price": 40,
      "earliest_time": "16:00",
      "min_spots": 2
    }
  ]
}
```

### Testing watchers before going live

```bash
# Show what would be sent right now, without POSTing to ntfy:
./.venv/bin/python alert.py --dry-run --verbose
```

By design, the very first run for each scrape date is silent (treated as
baseline) — otherwise you'd get a flood of "matches" on day one.

## If something breaks

Open `inspect_endpoint.md` for the debugging recipe. The most likely failure
modes are: ForeUp changing a `booking_class_id`, a course moving to a new
`course_id`, or a new authentication wall. None take more than a few minutes
to track down.
